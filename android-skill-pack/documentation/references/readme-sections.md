# README Sections

Use this reference when a project needs concise documentation.

## Good Default Sections

- What the app does.
- Tech stack and architectural pattern.
- How to build and run it.
- How to test it.
- Notable implementation decisions or constraints.

## Writing Guidance

- Keep each section short and skimmable.
- Prefer commands and file locations over broad narrative.
- Mention required environment assumptions if they affect setup.

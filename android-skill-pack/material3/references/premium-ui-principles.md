# Premium UI Principles

Use this reference when a request needs more visual direction than the main skill body should carry.

## Traits To Aim For

- Clear hierarchy: the primary action and most important content are obvious immediately.
- Calm surfaces: backgrounds and containers support the content instead of competing with it.
- Purposeful rhythm: spacing is consistent but not robotic.
- Confident typography: headings, labels, and supporting text have distinct jobs.
- Controlled color: most of the screen stays quiet so accents can matter.

## Common Generic UI Smells

- Equal emphasis on every card, button, and statistic.
- Repeated rounded rectangles with no hierarchy.
- Random icon colors and multiple unrelated accents.
- Tight spacing mixed with oversized headers.
- Decorative gradients that reduce readability.

## Tactics

- Use one dominant content axis and align sections to it.
- Let one element own the visual focus per viewport section.
- Prefer a small number of surface treatments reused consistently.
- Use typography changes more often than new container styles.
- If a screen feels busy, remove a treatment before adding another.
